# 7. 데모 런북 (진행자용 체크리스트)

MSP 대상 데모를 진행할 때 **위에서 아래로** 순서대로 따라가세요.
각 단계는 해당 상세 문서를 참조합니다.

## 사전 준비 (데모 시작 5분 전)
- [ ] `az login` 후 올바른 구독 선택
      `az account show -o table`
- [ ] 서버 기동:
      `az vm start -g rg-cyclecloud-training -n cc-server`
- [ ] 1~2분 대기 후 서비스 확인:
      ```bash
      az vm run-command invoke -g rg-cyclecloud-training -n cc-server \
        --command-id RunShellScript --scripts "systemctl is-active cycle_server" \
        --query "value[0].message" -o tsv
      ```
- [ ] 포털 접속 확인: https://cc-cyclecloud-ekwphu.koreacentral.cloudapp.azure.com

## 데모 시나리오 (약 40~60분)

### ① 포털 소개 + 최초 설정 — 5분 · [02](02-포털-사용법.md)
- [ ] (최초 1회) 설정 마법사: 라이선스 동의 → 관리자 계정 → 구독 등록
- [ ] 구독 등록 시 **Locker 인증 = Managed Identity**, 자격증명으로 `cc-locker-id` 선택
      (공유 키로 만들면 정책상 노드가 `Staging resources 403` 실패 → [06](06-트러블슈팅-로그.md) 6.4)
- [ ] Clusters / Nodes / Settings 메뉴 둘러보기

### ② 신규 클러스터 생성 — 10분 · [03](03-신규-클러스터-생성.md)
- [ ] Clusters → `+` → **Slurm** 선택
- [ ] 이름 `slurm-demo`, Region `Korea Central`, VM `Standard_D4s_v5`, Max 100 cores
- [ ] Networking: `cc-vnet` / `compute` 서브넷
- [ ] Save → **Start** → 스케줄러 노드가 `Ready` 되는 과정 설명

### ③ 노드 증설/감설 + 사이즈 변경 — 10분 · [04](04-노드-증감설-사이즈변경.md)
- [ ] Arrays 탭에서 `hpc` 배열 수동 증설(예: 2대) → 상태 관찰
- [ ] 작업 제출로 자동확장 시연(선택): 스케줄러 노드에서 `sbatch`
- [ ] Edit → VM Type 변경(D4s_v5→D8s_v5) → Terminate 후 재생성 설명
- [ ] 감설: 노드 Terminate

### ④ 스토리지/디스크 마운트 — 10분 · [05](05-스토리지-디스크-마운트.md)
- [ ] Locker 개념 설명(`cyclecloud locker list`)
- [ ] cloud-init 로 NFS/`Azure Files`/`blobfuse2` 마운트 예시 시연
- [ ] (선택) 노드에 추가 관리 디스크 붙이는 템플릿 설명

### ⑤ 트러블슈팅/로그 — 10분 · [06](06-트러블슈팅-로그.md)
- [ ] `az vm run-command` 로 `cycle_server.log` 확인(SSH 없이)
- [ ] `cyclecloud show_nodes -c slurm-demo` 상태 확인
- [ ] 오류 노드의 `jetpack.log` / `cloud-init-output.log` 위치 설명
- [ ] 자주 발생하는 문제표(쿼터/권한/약관) 리뷰

## 데모 종료 후 (⚠️ 과금 정리 필수)
- [ ] 클러스터 종료: 포털에서 `slurm-demo` → **Terminate**
      (또는 `cyclecloud terminate_cluster slurm-demo`)
- [ ] 서버 중지(deallocate):
      `az vm deallocate -g rg-cyclecloud-training -n cc-server`
- [ ] (완전 삭제 시) `az group delete -n rg-cyclecloud-training --yes`
