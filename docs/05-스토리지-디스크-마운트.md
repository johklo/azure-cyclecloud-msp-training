# 5. Storage Account / Disk 마운트

요청사항 **"Storage account, Disk 마운트"** 에 해당합니다.
HPC 클러스터에서 스토리지는 크게 3가지 목적으로 씁니다.

| 구분 | 예 | 용도 |
|------|-----|------|
| **로컬 디스크** | 노드 OS/임시 디스크, 추가 관리 디스크 | 노드별 임시/스크래치 |
| **공유 파일시스템(NFS)** | 스케줄러 노드 NFS, Azure NetApp Files, Azure Files | 모든 노드 공용 `/home`, `/shared` |
| **오브젝트 스토리지(Blob)** | Storage Account Blob (blobfuse2) | 대용량 입력/출력 데이터 |

---

## 5.1 노드에 추가 관리 디스크(Managed Disk) 붙이기
노드에 스크래치용 디스크를 추가하려면 **클러스터 템플릿의 `[[[volume]]]`** 를 사용합니다.

### 포털
1. **Clusters → 클러스터 → Edit → (배열의) Storage / Attached Volumes**.
2. 디스크 크기(GB), 개수, 유형(SSD/Premium) 지정 → Save → 노드 재생성 시 반영.

### 템플릿(.txt) 예시
```ini
[[node defaults]]
    [[[volume scratch]]]
    Size = 128           # GB
    SSD = true           # Premium SSD
    Mount = scratch      # /mnt/scratch 등으로 마운트 스펙 연결

    [[[configuration cyclecloud.mounts.scratch]]]
    mountpoint = /mnt/scratch
    fs_type = ext4
```
> 관리 디스크는 노드 생성 시 attach → 포맷/마운트됩니다. 노드 종료 시 삭제되므로
> **영구 데이터는 공유 파일시스템/Blob 에 저장**하세요.

---

## 5.2 공유 파일시스템(NFS) 마운트
모든 노드가 공유하는 `/home`, `/shared` 는 보통 **NFS** 로 제공합니다.

### (A) 내장(Built-in) NFS — 스케줄러 노드가 NFS 서버
포털 → 클러스터 **Edit → Network Attached Storage** 에서:
- `NFS Type = Builtin` (스케줄러 노드가 `/shared`, `/home` export)
- 실습/소규모에 적합. 대규모·고성능은 아래 관리형 서비스 권장.

### (B) 외부 NFS / Azure NetApp Files
- `NFS Type = External NFS` 선택 후 **NFS 서버 IP / Export 경로** 입력.
- 예: Azure NetApp Files 볼륨(`10.0.2.4:/hpcvol`) 을 `/shared` 로 마운트.

### 수동 마운트(cluster-init/cloud-init 예)
```bash
sudo mkdir -p /shared
sudo mount -t nfs 10.0.2.4:/hpcvol /shared
echo "10.0.2.4:/hpcvol /shared nfs defaults 0 0" | sudo tee -a /etc/fstab
```

## 5.3 Azure Files(SMB) 마운트
Storage Account 의 **파일 공유(File Share)** 를 SMB 로 마운트:
```bash
sudo mkdir -p /mnt/azfiles
STORAGE=cclkekwphusd3i
SHARE=hpcshare
# 스토리지 계정 키는 Key Vault/관리 ID 사용 권장 (아래는 예시)
sudo mount -t cifs //$STORAGE.file.core.windows.net/$SHARE /mnt/azfiles \
  -o vers=3.0,username=$STORAGE,password=$STORAGE_KEY,dir_mode=0777,file_mode=0777,serverino
```
> 클러스터 전 노드에 적용하려면 이 스크립트를 **cluster-init 프로젝트** 또는
> 노드 배열의 `cloud-init` 에 넣습니다(5.5).

---

## 5.4 Storage Account(Blob) 마운트 — blobfuse2
대용량 데이터를 Blob 컨테이너로 마운트(POSIX 처럼 접근):
```bash
# blobfuse2 설치 (HPC 이미지엔 보통 포함)
sudo mkdir -p /mnt/blob /mnt/blobcache
cat <<'YAML' | sudo tee /etc/blobfuse2.yaml
allow-other: true
components: [libfuse, file_cache, attr_cache, azstorage]
file_cache: { path: /mnt/blobcache }
azstorage:
  type: block
  account-name: cclkekwphusd3i
  container: cyclecloud
  mode: msi          # VM 관리 ID 사용 (키 불필요, 권장)
YAML
sudo blobfuse2 mount /mnt/blob --config-file=/etc/blobfuse2.yaml
```
> `mode: msi` 를 쓰려면 노드/서버 관리 ID 에 해당 스토리지의
> **Storage Blob Data Contributor** 역할이 필요합니다(azure-rbac 참고).

## 5.5 클러스터 전체에 자동 적용 (cluster-init / cloud-init)
위 마운트를 **모든 노드에 자동 적용**하려면 두 가지 방법이 있습니다.

**1) cloud-init (간단):** 포털 → 클러스터 **Edit → Advanced → Cloud-init** 에
배열별로 셸 스크립트를 입력하면 노드 부팅 시 실행됩니다.
```yaml
#cloud-config
runcmd:
  - mkdir -p /shared
  - mount -t nfs 10.0.2.4:/hpcvol /shared
```

**2) cluster-init 프로젝트 (재사용/버전관리):**
```bash
cyclecloud project init mymounts       # 프로젝트 생성
# specs/default/cluster-init/scripts/00-mount.sh 에 마운트 스크립트 작성
cyclecloud project upload <locker>     # Locker(cclkekwphusd3i)로 업로드
# 클러스터 Edit → 배열의 "Cluster-Init" 에 mymounts:default 지정
```

## 5.6 Locker(프로젝트 스토리지) 개념
`cclkekwphusd3i` 스토리지 계정의 `cyclecloud` 컨테이너가 **Locker** 입니다.
cluster-init 프로젝트/블롭이 여기에 저장되어 노드가 부팅 시 내려받습니다.
```bash
cyclecloud locker list          # Locker 확인
cyclecloud project upload       # 프로젝트 업로드
```
