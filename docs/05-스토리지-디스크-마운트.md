# 5. Storage Account / Disk 마운트

이 문서는 요청사항 **"Storage account, Disk 마운트"**에 해당하며, HPC 노드의 OS 디스크 사이즈 변경 규칙, 공유 파일시스템(NFS/Lustre), Blob 마운트(`blobfuse2`), 그리고 **Locker 프로젝트 스토리지** 개념을 다룹니다.

---

## 5.1 HPC 노드 OS 디스크 사이즈 변경

CycleCloud 노드는 VMSS로 프로비저닝됩니다. OS 디스크 기본값은 보통 64GB로 생성되나, KT 환경에서는 **노드별 OS Disk 1TB** 등을 지정하여 사용합니다.

### 1) UI에서 BootDiskSize 변경
![BootDiskSize 변경 UI](images/disk-resize/1780359188335.png)

1. **Clusters → 클러스터 선택 → Edit → Advanced Settings**.
2. **BootDiskSize** 항목을 원하는 크기(예: `128` 또는 `1000` GB)로 변경.
3. **Save** 클릭.

### 2) 클러스터 재시작 (Terminate 후 Start)
![Terminate 후 Start](images/disk-resize/1780359220774.png)

> ⚠️ **CRITICAL WARNING: OS 디스크 변경 시 전체 재시작 필수**  
> 운영 중인 클러스터에서 UI의 `BootDiskSize`만 변경하고 기존 노드를 그대로 유지하면, 새로 뜨는 노드가 동일 VMSS 내에서 기존 노드와 **디스크 용량 불일치 오류**(`VMSS OS disk size mismatch`)가 발생하여 프로비저닝이 실패합니다.  
> 반드시 **클러스터 전체를 Terminate 후 Start** 하여 VMSS를 재생성해야 합니다.

---

## 5.2 공유 파일시스템 (NFS / Lustre) 마운트

HPC 클러스터에서는 모든 노드가 동일한 데이터를 참조할 수 있도록 공유 파일시스템을 마운트합니다.

| 종류 | KT 구성 예시 | 용도 및 특징 |
|------|--------------|--------------|
| **Built-in NFS** | Master 노드가 `/shared`, `/home` export | 실습 및 기본 데이터 공유 |
| **Blob Storage NFS** | 2개 컨테이너 NFS 마운트 | 대용량 입출력 파일 보관 |
| **Azure Managed Lustre** | **512TB** 초고속 병렬 파일시스템 마운트 | 초고성능 GPU 학습/분석 데이터셋 |

## 5.3 Azure Files (SMB) & Blob Storage (`blobfuse2`) 마운트

### 1) Azure Files (SMB/CIFS)
```bash
sudo mkdir -p /mnt/azfiles
STORAGE="cclkekwphusd3i"
SHARE="hpcshare"

sudo mount -t cifs //$STORAGE.file.core.windows.net/$SHARE /mnt/azfiles \
  -o vers=3.0,username=$STORAGE,password=$STORAGE_KEY,dir_mode=0777,file_mode=0777,serverino
```

### 2) blobfuse2 (Azure Blob Storage POSIX 마운트)
VM 관리 ID(Managed Identity)를 활용하여 시크릿 키 없이 Blob 컨테이너를 마운트합니다:
```bash
sudo mkdir -p /mnt/blob /mnt/blobcache

cat <<'YAML' | sudo tee /etc/blobfuse2.yaml
allow-other: true
components: [libfuse, file_cache, attr_cache, azstorage]
file_cache: { path: /mnt/blobcache }
azstorage:
  type: block
  account-name: cclkekwphusd3i
  container: cyclecloud
  mode: msi          # VM 관리 ID 사용
YAML

sudo blobfuse2 mount /mnt/blob --config-file=/etc/blobfuse2.yaml
```
*(참고: 관리 ID 사용 시 해당 Storage Account에 `Storage Blob Data Contributor` RBAC 역할이 부여되어 있어야 합니다.)*

## 5.4 Locker (프로젝트/템플릿 스토리지) 개념

CycleCloud에서 **Locker**는 클러스터 템플릿, 노드 구성 파일, 그리고 `cluster-init` 스크립트가 저장되는 Azure Blob Storage 컨테이너입니다.

- **스토리지 계정**: `cclkekwphusd3i`
- **컨테이너**: `cyclecloud`
- **Locker 확인 명령어**:
  ```bash
  cyclecloud locker list
  # OUTPUT:
  # cyclecloud-lab-storage (az://cclkekwphusd3i/cyclecloud)
  ```
