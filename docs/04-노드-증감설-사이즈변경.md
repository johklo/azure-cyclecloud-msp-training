# 4. 노드 증설/감설 및 노드 사이즈 변경

요청사항 **"Cycle cloud 노드 증/감설, 노드 사이즈 변경"** 에 해당합니다.

CycleCloud 에서 노드는 **노드 배열(node array)** 단위로 관리됩니다
(예: `scheduler`, `hpc`(=execute), `htc`, `login`).

## 4.1 노드 증설/감설 방식 2가지
| 방식 | 동작 | 사용 상황 |
|------|------|-----------|
| **자동확장(Autoscale)** | 제출된 작업(Job) 수요에 따라 실행 노드를 자동 증감 | 평상시 권장 |
| **수동(Manual)** | 운영자가 노드 수를 직접 지정 | 테스트/사전 워밍업 |

### (A) 자동확장 상한 조정 — 포털
1. 포털 → **Clusters → slurm-train → Edit**.
2. **Required Settings** 에서 `Max HPC Cores`(또는 Max Nodes) 값을 변경.
   - 예: 100 → 200 (증설 상한 확대) / 100 → 40 (감설 상한 축소)
3. **Save** → 클러스터에 설정 적용(필요 시 재시작). 이후 작업량에 따라 자동 증감.

### (B) 수동 증설/감설 — 포털
1. 포털 → **Clusters → slurm-train → Arrays 탭** → `hpc`(execute) 배열 선택.
2. **Scale** (또는 "Add Nodes") → 원하는 노드 수 입력 → 적용.
   - 증설: 노드 수를 늘리면 즉시 VM 프로비저닝 시작(`Acquiring→Ready`).
   - 감설: 노드를 선택 → **Terminate** 로 개별/일괄 종료.

### (C) 수동 증설/감설 — CLI
서버에서(5장 접근 방법):
```bash
# 현재 노드 상태 확인
cyclecloud show_nodes -c slurm-train

# 실행 노드 5대 추가 (배열 이름은 템플릿에 따라 hpc/execute 등)
cyclecloud add_node slurm-train -t hpc -c 5

# 특정 노드 종료(감설)
cyclecloud terminate_node slurm-train -n hpc-1

# 배열 전체를 0대로 (모든 실행 노드 종료)
cyclecloud terminate_node slurm-train -t hpc
```
> Slurm 자동확장은 `azslurm`/`cyclecloud_slurm` 연동으로 동작합니다. 작업을 제출하면
> (`sbatch job.sh`) 필요한 만큼 노드가 자동 생성되고, 유휴 시 자동 종료됩니다.

## 4.2 노드 사이즈(VM SKU) 변경
CycleCloud 는 실행 중인 노드의 크기를 "실시간"으로 바꾸지 않습니다.
**노드 배열의 Machine Type 을 변경 → 기존 노드 종료 → 새 크기로 재생성** 방식입니다.

### 포털
1. 포털 → **Clusters → slurm-train → Edit**.
2. 변경할 배열의 **VM Type / Machine Type** 을 원하는 SKU 로 선택.
   - 예: `Standard_D4s_v5` → `Standard_D8s_v5` (또는 HPC 계열 `Standard_HB176rs_v4`).
3. **Save**.
4. 해당 배열의 **기존 실행 노드를 Terminate** → 다음 증설/작업 시 **새 크기**로 생성됨.
   (스케줄러 노드 크기를 바꾼 경우 스케줄러 노드를 재시작해야 반영)

### CLI
```bash
# 배열의 머신 타입 변경 후 노드 재생성
cyclecloud modify_cluster slurm-train ...   # 또는 Edit 후 파라미터 재적용
cyclecloud terminate_node slurm-train -t hpc   # 기존 노드 종료
cyclecloud add_node slurm-train -t hpc -c 2     # 새 크기로 증설
```

### 사이즈 변경 시 주의사항
- **쿼터(Quota)**: 큰/특수 SKU(예: HB, ND GPU)는 리전별 vCPU 쿼터가 별도입니다.
  변경 전 `az vm list-usage -l koreacentral` 로 확인하고, 부족하면 쿼터 증설 요청.
- **가용성**: 특정 SKU 가 리전/영역에 없을 수 있음 → `az vm list-skus -l koreacentral --size Standard_H` 로 확인.
- **HPC 계열**(HB/HC/ND)은 InfiniBand/드라이버가 필요하므로 HPC용 OS 이미지 사용.
- 데이터 유실 방지: 노드 로컬 디스크 데이터는 종료 시 사라집니다 → 영구 데이터는 5장의 공유 스토리지에 저장.

## 4.3 확인
```bash
cyclecloud show_nodes -c slurm-train      # 노드 수/상태/크기 확인
```
포털 **Arrays** 탭에서 노드 수와 각 노드의 VM 크기를 확인합니다.
