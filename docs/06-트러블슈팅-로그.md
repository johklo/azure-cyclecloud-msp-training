# 6. 기본 트러블슈팅 및 로그 확인

요청사항 **"기본적인 트러블슈팅 방법 (로그 확인 방법)"** 에 해당합니다.

## 6.1 서버 접근 방법 (로그를 보려면)
CycleCloud 서버(`cc-server`)에 접근하는 방법은 3가지입니다.

### (A) Azure Portal / az CLI "명령 실행" — SSH 키 불필요 ✅ 권장
Azure RBAC 권한만 있으면 됩니다.
- 포털: **VM `cc-server` → 운영 → 명령 실행(Run command) → RunShellScript** 에 명령 입력.
- CLI:
  ```bash
  az vm run-command invoke -g rg-cyclecloud-training -n cc-server \
    --command-id RunShellScript \
    --scripts "systemctl is-active cycle_server; tail -n 50 /opt/cycle_server/logs/cycle_server.log" \
    --query "value[0].message" -o tsv
  ```

### (B) SSH
```bash
ssh -i keys/cyclecloud_rsa azureadmin@20.196.213.145
```
> SSH 는 NSG 에서 허용된 소스 IP 에서만 접속됩니다(`cc-nsg` 의 `Allow-SSH` 규칙).
> 접속이 안 되면 6.5 의 NSG 확인 절차를 보세요.
>
> 새 담당자 PC 에서 접속하려면: ① `ssh-keygen -t rsa -b 4096 -f mykey -N ""` 로
> 키를 만들고 ② `az vm user update -g rg-cyclecloud-training -n cc-server -u azureadmin
> --ssh-key-value "$(cat mykey.pub)"` 로 공개키를 등록한 뒤 ③ 그 담당자 IP 를
> `cc-nsg` 의 `Allow-SSH` 소스에 추가하세요. (키 없이도 (A) 방법으로 로그 확인 가능)

### (C) Azure Serial Console / Bastion
포털 → VM → **직렬 콘솔(Serial console)** 또는 **Bastion** 으로도 접근 가능(로컬 계정 필요).

## 6.2 CycleCloud 서버 로그 위치 (검증됨)
경로: **`/opt/cycle_server/logs/`**

| 로그 파일 | 내용 |
|-----------|------|
| `cycle_server.log` | **메인 애플리케이션 로그** (가장 먼저 확인) |
| `catalina.out` / `catalina.err` | 웹/서비스(Tomcat) 기동·오류 로그 |
| `installation.log` | 서버 설치/초기화 로그 |
| `imds.log.log` | 관리 ID(IMDS) 토큰 관련 로그 |

자주 쓰는 확인 명령:
```bash
sudo systemctl status cycle_server            # 서비스 상태
sudo tail -n 100 -f /opt/cycle_server/logs/cycle_server.log   # 실시간 로그
sudo grep -iE "error|exception|denied" /opt/cycle_server/logs/cycle_server.log | tail -50
sudo systemctl restart cycle_server           # 서비스 재시작
```

## 6.3 클러스터 / 노드 상태 및 로그
### 포털
- **Clusters → 클러스터 → Arrays** 에서 노드 상태 색상/메시지 확인.
- 문제 노드 클릭 → **상태 메시지 / "Errors"** 확인 (예: 쿼터, 이미지, 부팅 실패).

### CLI (서버에서)
```bash
cyclecloud show_cluster <cluster>        # 클러스터 요약
cyclecloud show_nodes -c <cluster>       # 노드별 상태(Status/State)
cyclecloud show_nodes -c <cluster> --states=Error   # 오류 노드만
```

### 개별 계산 노드 내부 로그 (부팅/프로비저닝 실패 시)
노드에는 **jetpack** 에이전트 로그가 있습니다.
```bash
cyclecloud connect <node> -c <cluster>   # 노드로 SSH
sudo cat /opt/cycle/jetpack/logs/jetpack.log      # 노드 초기화 로그
sudo cat /var/log/cloud-init-output.log           # cloud-init/스크립트 로그
```

## 6.4 자주 발생하는 문제와 해결
| 증상 | 원인 | 해결 |
|------|------|------|
| 노드가 `Error`/`Failed`, "Quota" 메시지 | vCPU 쿼터 초과 | `az vm list-usage -l koreacentral` 확인 → 쿼터 증설 요청 또는 작은 SKU |
| 노드가 `Staging resources (403)` 에서 실패 | Locker 가 공유 키 인증(정책상 차단) 사용 | Locker 를 **관리 ID** 로 전환 + Blob **프라이빗 엔드포인트** 필요(6.5) |
| 노드가 `CreateVirtualMachines ... ResourceId is not valid` | Locker 관리 ID 값이 클라이언트 ID | Locker 관리 ID 를 사용자 할당 ID 의 **리소스 ID** 로 지정 |
| 노드가 계속 `Acquiring`/생성 안 됨 | 관리 ID 권한 부족 | 서버 관리 ID 에 RG `Contributor` 있는지 확인(6.5) |
| "Marketplace terms not accepted" | 이미지 약관 미동의 | `az vm image terms accept --urn <urn>` |
| 노드 부팅 후 스케줄러 미등록 | cluster-init/네트워크 문제 | 노드의 `jetpack.log`, `cloud-init-output.log` 확인 |
| 포털 접속 불가(HTTPS) | 서비스 다운 or NSG | `systemctl status cycle_server`, NSG 443 확인(6.5) |
| Spot 노드가 사라짐 | Spot 축출(eviction) | Spot→일반 우선순위 변경 또는 재시도 |
| Locker/프로젝트 업로드 실패(403) | Locker 관리 ID 의 Blob 권한/프라이빗 엔드포인트 문제 | `cc-locker-id` 에 `Storage Blob Data Contributor`, Blob PE/DNS 확인(6.5) |

## 6.5 인프라(RBAC/NSG) 확인 명령 (az CLI)
```bash
# 관리 ID(Principal) 확인
PID=$(az vm show -g rg-cyclecloud-training -n cc-server \
  --query identity.principalId -o tsv)

# 이 관리 ID 의 역할 할당 확인 (RG Contributor 있어야 함)
az role assignment list --assignee $PID -o table

# Locker 관리 ID(cc-locker-id) 의 Blob 데이터 권한 확인
LID=$(az identity show -g rg-cyclecloud-training -n cc-locker-id \
  --query principalId -o tsv)
az role assignment list --assignee $LID -o table   # Storage Blob Data Contributor 있어야 함

# Blob 프라이빗 엔드포인트/DNS 확인 (서버에서 사설 IP 로 해석되어야 함)
az network private-endpoint list -g rg-cyclecloud-training -o table
# 서버 내부: getent hosts <storage>.blob.core.windows.net  → 10.0.0.x 여야 정상

# 계산 노드용 vCPU 쿼터 확인
az vm list-usage -l koreacentral -o table | grep -iE "vCPU"

# NSG 규칙 확인 (443/22 인바운드)
az network nsg rule list -g rg-cyclecloud-training --nsg-name cc-nsg -o table

# 특정 SKU 가용성 확인
az vm list-skus -l koreacentral --size Standard_D8s --all -o table
```

## 6.6 지원 에스컬레이션
위 절차로 해결이 안 되면 로그(`cycle_server.log`, 노드 `jetpack.log`)와
재현 절차를 첨부하여 Azure SRE 팀(이동훈/김태진/지민근)에 문의하세요.
