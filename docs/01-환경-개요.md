# 1. 환경 개요 및 아키텍처

## 1.1 CycleCloud 란?
Azure CycleCloud 는 Azure 위에서 **HPC(고성능 컴퓨팅) 클러스터**를 손쉽게 만들고
운영·자동확장(Autoscale)할 수 있게 해 주는 **오케스트레이션 도구**입니다.

- 스케줄러(Slurm, OpenPBS, LSF, HTCondor, Grid Engine 등) 클러스터를 템플릿으로 생성
- 작업(Job) 부하에 따라 계산 노드를 **자동 증설/감설**
- 관리자용 **웹 포털** + **CLI(`cyclecloud`)** + **REST API** 제공

## 1.2 구성 요소
```
                    ┌─────────────────────────────────────────┐
   운영자/브라우저 ─▶│  CycleCloud 서버 VM (cc-server)          │
   (HTTPS 443)      │   - 웹 포털 / cycle_server 서비스         │
                    │   - cyclecloud CLI                       │
                    │   - System-assigned MI ─▶ 노드 생성(RG Contributor)
                    │   - User-assigned MI (cc-locker-id) ─▶ Locker 접근
                    └───────────────┬─────────────────────────┘
                                    │ 클러스터 프로비저닝
                     ┌──────────────┼───────────────────────┐
                     ▼              ▼                       ▼
              스케줄러 노드     실행(Execute) 노드 …    (Autoscale)
                     └──────── VNet: cc-vnet / subnet: compute ──────┘

   Storage Account(Locker) : 프로젝트/블롭 저장소
     - 공용 접근 차단 → Blob 프라이빗 엔드포인트로 접근
     - 관리 ID(cc-locker-id) 인증 (공유 키 미사용)
```

| NSG | `cc-nsg` | 443(포털)/80/22(SSH) 인바운드 허용 |

## 1.5 권한(RBAC) 개념
CycleCloud 서버 VM 은 두 개의 관리 ID 를 사용합니다.
- **시스템 할당 관리 ID**: 리소스 그룹 범위 `Contributor` → 노드 VM 생성/삭제.
- **사용자 할당 관리 ID(`cc-locker-id`)**: `Storage Blob Data Contributor` → Locker(스토리지) 접근.
  이 ID 는 서버와 **모든 계산 노드에 함께 연결**되어, 공유 키 없이 Locker 를 읽고 씁니다.

→ 그래서 CycleCloud 는 자격 증명(암호/시크릿) 저장 없이 노드 생성과 Locker 접근을 수행합니다.
운영 환경에서도 이 관리 ID 기반(키 미사용) 구성을 권장합니다.

> ℹ️ 이 구독은 Azure Policy 로 스토리지 **공용 접근·공유 키가 차단**됩니다. 그래서 Locker 는
> 반드시 관리 ID 로 접근하며, 서버·노드가 도달하도록 **Blob 프라이빗 엔드포인트**가 배포됩니다.
