# 1. 환경 개요 및 아키텍처

**Azure CycleCloud**의 주요 개념, 시스템 아키텍처, **KT CycleCloud 환경 구축 현황 및 운영 지침**을 정리합니다.

---

## 1.1 Azure CycleCloud 란?

**Azure CycleCloud**는 Azure에서 HPC 클러스터를 생성, 관리, 오케스트레이션(자동 증설/감설)하는 **엔터프라이즈 관리 서비스**입니다.

- **다양한 스케줄러 지원**: Slurm, OpenPBS, LSF, HTCondor, Grid Engine 등
- **오토스케일링(Autoscale)**: 작업(Job) 큐의 수요에 따라 VMSS 기반 계산 노드를 자동으로 증설 및 감설
- **통합 제어 인터페이스**: 웹 포털 GUI, CLI(`cyclecloud`), REST API 제공

---

## 1.2 시스템 구성 아키텍처

```
                    ┌─────────────────────────────────────────┐
   운영자/브라우저 ─▶│  CycleCloud 서버 VM (cc-server)          │
   (HTTPS 443)      │   - 웹 포털 / cycle_server 서비스         │
                    │   - cyclecloud CLI                       │
                    │   - System-assigned Managed Identity     │──▶ 구독 범위 노드 생성
                    └───────────────┬─────────────────────────┘   (Contributor)
                                    │ 클러스터 프로비저닝 (CLI / Template)
                     ┌──────────────┼───────────────────────┐
                     ▼              ▼                       ▼
              스케줄러 노드     실행(Execute) 노드 …    (Autoscale)
              (slurm-master)  (VMSS 기반 계산 노드)
                     └──────── VNet: cc-vnet / subnet: compute ──────┘

   Storage Account (cclkekwphusd3i) = "Locker" : 프로젝트/템플릿/cluster-init 객체 저장소
```

### 주요 구성 요소
1. **Control Plane (CycleCloud Server VM)**:
   - `cc-server`에서 `cycle_server`를 실행하고 Azure 리소스를 프로비저닝합니다.
   - Azure 관리 ID(Managed Identity)를 사용하므로 시크릿/암호가 필요하지 않습니다.
2. **Scheduler Node (Master Node)**:
   - `slurmctld`로 워크로드를 스케줄링하고 NFS(`/shared`, `/sched`)를 제공합니다.
3. **Execute Nodes (Worker Nodes)**:
   - 작업(Job)을 수행하는 계산 전용 VM입니다.
   - 작업 제출 시 자동 생성되고 유휴 시 자동 종료됩니다.
4. **Locker (Blob Storage)**:
   - 클러스터 템플릿과 `cluster-init` 스크립트를 보관합니다.

## 1.6 네트워크·보안 아키텍처 (개념)

### 1.6.1 서브넷 구조 — 서버와 계산 노드의 분리

CycleCloud 환경은 하나의 VNet(`cc-vnet`, `10.0.0.0/16`) 안에서 역할별 서브넷을 분리합니다.

| 서브넷 | 대역 | 용도 | 분리 이유 |
|--------|------|------|-----------|
| **서버 서브넷** (`cyclecloud`) | `10.0.0.0/24` | CycleCloud Server VM 전용 | 제어 평면(Control Plane). 포털/오케스트레이션만 담당하므로 NSG를 좁게(443/22) 유지 |
| **계산 서브넷** (`compute`) | `10.0.1.0/24` | 스케줄러·계산 노드(VMSS) 전용 | 데이터 평면(Data Plane). 노드가 **대량·수시로 생성/삭제**되므로 서버와 분리해 NSG·라우팅을 독립 관리 |
| **AzureBastionSubnet** | `/26` 이상 | Bastion 전용 (선택) | Bastion 사용 시 **이름·크기 고정 규격**이 강제됨(`/26` 이상, 이름 변경 불가) |

**IP 사이징 주의**
- 계산 노드는 **노드 1대당 IP 1개**를 계산 서브넷에서 사용합니다. `/24`(약 251개 가용)는 대략 **250노드**가 한계입니다.
- KT 운영(Slurm 100노드)처럼 규모가 크거나 증설 가능성이 있으면 계산 서브넷을 **`/23`·`/22`로 넉넉히** 설계하십시오.
- 클러스터 생성 시 서브넷 선택 위치는 [3장 §3.3 Required Settings](03-신규-클러스터-생성.md)를 참고하세요.

### 1.6.2 Private Endpoint & 보안 (스토리지/DB)

보안 강화 환경에서는 스토리지와 DB를 VNet 내부 사설 IP로만 접근하도록 구성합니다.

- **스토리지(Locker) Private Endpoint**: Shared Key(계정 키)를 비활성화하는 정책을 적용하면(CycleCloud 8.7+ 권장), 노드/서버는 **Managed Identity 로만** Blob에 접근합니다. 공용 엔드포인트를 차단하고 **Private Endpoint(예: `cc-blob-pe`, `10.0.0.5`) + Private DNS Zone**(`privatelink.blob.core.windows.net`)을 구성합니다.
  - **Private DNS 연결 누락 시** 노드가 공용 IP로 이름을 해석하여 다운로드 403/타임아웃이 발생합니다. **PE + Private DNS 는 세트**입니다.
  - MI 권한 부여는 [3장 신규 클러스터 생성](03-신규-클러스터-생성.md)의 Managed Identity/RBAC 설명을 참고하세요.
- **DB(Job Accounting) Private Access**: MySQL Flexible Server는 **Private Access (VNet Integration)** 로 공용 IP를 차단하고 VNet에서만 접근합니다. 설정은 [7장 §7.1](07-Job-Accounting-설정.md)을 참고하세요.

### 1.6.3 Public IP & 접근 모델

접근 경로는 보안 요구 수준에 따라 선택합니다. 두 가지 Public IP 개념을 구분합니다.

| 구분 | 대상 | 역할 | 트레이드오프 |
|------|------|------|--------------|
| **서버 공인 IP** (`cc-pip`) | CycleCloud Server | 웹 포털(HTTPS 443) 접근 | 편리하지만 포털이 인터넷에 노출 → NSG로 출발지 IP 제한 권장. 운영 환경은 **PE/사설 + Bastion** 로 대체 가능 |
| **Scheduler Public IP** (클러스터 옵션) | 스케줄러 노드 | 포털을 거치지 않고 노드에 직접 SSH | 편의를 위해 계산 노드에 공용 IP 부여 → **미사용(Unchecked) 권장**. 대신 서버를 점프호스트로 사용 |

**Public IP 미사용(사설 전용) 시 접근 방법**
- **Azure Bastion**: 공용 IP 없이 포털에서 브라우저로 SSH/RDP. 현재 랩 환경이 Bastion 방식입니다.
- **점프호스트(서버 경유)**: CycleCloud 서버에 먼저 접속한 뒤 서버 내장 키로 계산 노드에 SSH. 노드 트러블슈팅은 [11장 §11.x](11-트러블슈팅-로그.md)를 참고하세요.

**보안 등급별 권장 구성 (참고)**

| 등급 | 서버 접근 | 노드 접근 | 스토리지 |
|------|-----------|-----------|----------|
| **개발/학습** | 공인 IP + NSG 출발지 제한 | Scheduler Public IP 허용 가능 | Shared Key 허용 가능 |
| **운영(권장)** | 사설 + **Bastion** | 서버 점프호스트만 | **Shared Key 비활성화 + PE + MI** |
