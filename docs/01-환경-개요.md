# 1. 환경 개요 및 아키텍처

## 1.1 CycleCloud 란?
Azure CycleCloud 는 Azure 위에서 **HPC(고성능 컴퓨팅) 클러스터**를 손쉽게 만들고
운영·자동확장(Autoscale)할 수 있게 해 주는 **오케스트레이션 도구**입니다.

- 스케줄러(Slurm, OpenPBS, LSF, HTCondor, Grid Engine 등) 클러스터를 템플릿으로 생성
- 작업(Job) 부하에 따라 계산 노드를 **자동 증설/감설**
- 관리자용 **웹 포털** + **CLI(`cyclecloud`)** + **REST API** 제공

## 1.2 구성 요소
```
                    ┌─────────────────────────────────────────┐
   운영자/브라우저 ─▶│  CycleCloud 서버 VM (cc-server)          │
   (HTTPS 443)      │   - 웹 포털 / cycle_server 서비스         │
                    │   - cyclecloud CLI                       │
                    │   - System-assigned Managed Identity     │──▶ 구독에 노드 생성
                    └───────────────┬─────────────────────────┘   (Contributor)
                                    │ 클러스터 프로비저닝
                     ┌──────────────┼───────────────────────┐
                     ▼              ▼                       ▼
              스케줄러 노드     실행(Execute) 노드 …    (Autoscale)
                     └──────── VNet: cc-vnet / subnet: compute ──────┘

   Storage Account(cclkekwphusd3i) = "Locker" : 프로젝트/블롭 저장소
```

## 1.5 권한(RBAC) 개념
CycleCloud 서버 VM 은 **시스템 할당 관리 ID(Managed Identity)** 를 가지며,
이 ID 에 **구독 범위 Contributor** 역할이 부여되어 있습니다.
→ 그래서 CycleCloud 는 자격 증명(암호/시크릿) 저장 없이 노드 VM 을 생성/삭제할 수 있습니다.
운영 환경에서는 필요 최소 권한(예: 전용 리소스 그룹 범위)으로 축소하는 것을 권장합니다.
