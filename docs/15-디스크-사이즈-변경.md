# 15. 디스크 사이즈 변경 (OS/부팅 디스크)

CycleCloud 계산 노드는 **VMSS(가상 머신 확장 집합)** 로 생성됩니다. OS(부팅) 디스크 크기 변경은 VM SKU 변경([4.5](04-노드-증감설-사이즈변경.md))과 별개입니다.

> ⚠️ **변경 후 반드시 재시작**: `BootDiskSize`만 바꾸고 기존 노드를 유지하면, 같은 VMSS로 새로 뜨는 노드에서 `VMSS OS disk size mismatch`가 발생합니다. 스케줄러 노드도 디스크 변경 후 재시작해야 반영됩니다.

---

## 15.1 방식 ① 클러스터 전체 반영 (권장)

![BootDiskSize 변경](images/disk-resize/1780359188335.png)

1. **Clusters → 클러스터 → Edit → Advanced Settings**.
2. **BootDiskSize** 를 원하는 크기로 변경.
   - ⚠️ 기본값 `0` 은 실제 **64GB** 로 생성됩니다. 예: `128` → 128GB, `1000` → 1TB.
3. **Save** 후 클러스터 **Terminate → Start** 로 전체 재생성.

![Terminate 후 Start](images/disk-resize/1780359220774.png)

---

## 15.2 방식 ② 특정 파티션만 변경 (전체 재시작 없이)

특정 파티션(예: GPU 노드)의 OS 디스크만 바꿀 때는 해당 VMSS 를 직접 수정합니다. **새로 뜨는 노드에만** 반영됩니다.

1. 대상 파티션의 **VMSS 이름** 을 노드 리소스 그룹에서 확인.
2. 디스크 크기 변경:
   ```bash
   az vmss update -g <resource-group> -n <VMSS-name> \
     --set "virtualMachineProfile.storageProfile.osDisk.diskSizeGb=256"
   ```
3. 출력에서 `diskSizeGb` 변경을 확인합니다(기존 실행 노드는 영향 없음).

> 실행 중 노드는 유지하고 자동확장으로 새로 뜨는 노드에만 큰 디스크가 필요한 경우(예: GPU 데이터 캐시)에 사용합니다.
