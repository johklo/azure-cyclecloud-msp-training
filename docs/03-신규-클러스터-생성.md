# 3. CycleCloud 신규 클러스터 생성

이 장은 요청사항 **"Cycle cloud 신규 생성"** 에 해당합니다.
가장 널리 쓰이는 **Slurm** 클러스터를 예로 듭니다. (OpenPBS/LSF 등도 절차 동일)

## 3.1 포털에서 클러스터 생성 (권장)

1. 포털 → **Clusters** → 좌측 하단 **`+`** (또는 "New Cluster").
2. 스케줄러 템플릿 선택 → **Slurm** 클릭.
3. **About / Required Settings** 탭 입력:
   | 항목 | 예시 값 | 설명 |
   |------|---------|------|
   | Cluster Name | `slurm-train` | 클러스터 이름 |
   | Region | `Korea Central` | 배포 리전 |
   | Scheduler VM Type | `Standard_D4s_v5` | 스케줄러(헤드) 노드 크기 |
   | HPC VM Type | `Standard_D4s_v5` (실습) / `Standard_HB176rs_v4` (실운영 HPC) | 계산 노드 크기 |
   | Max HPC Cores | `100` | 실습은 작게 (쿼터 내) |
   | HTC VM Type / Max Cores | (선택) | 저우선순위/분산 작업용 |
4. **Networking** 탭:
   - Virtual Network: `cc-vnet`
   - Subnet: **`compute` (10.0.1.0/24)** 선택
5. **Advanced Settings** 탭(선택):
   - OS: 기본 이미지(예: AlmaLinux/Ubuntu HPC) 유지
   - Scheduler에 Public IP 부여 여부, 스토리지(다음 5장) 등
6. **Save** → 목록에 `slurm-train` 이 **(중지됨)** 상태로 생성됨.
7. 클러스터 선택 → **Start** 클릭 → 스케줄러(헤드) 노드부터 프로비저닝 시작.
   - 상태가 `Acquiring → Preparing → Ready` 순서로 진행됩니다(수 분 소요).

> 실행(계산) 노드는 기본적으로 **작업이 제출될 때 자동 증설(autoscale)** 됩니다.
> 수동 증설은 4장을 참고하세요.

## 3.2 CLI 로 클러스터 생성 (자동화용)
서버 접속 후(SSH 또는 `az vm run-command`, 5장):
```bash
# 1) 사용 가능한 템플릿 확인
cyclecloud show_cluster --help
# 2) 파라미터 파일로 클러스터 임포트
cyclecloud import_cluster slurm-train -c Slurm -f /opt/cycle_server/templates/slurm.txt \
  -p params.json
# 3) 시작 / 상태 확인
cyclecloud start_cluster slurm-train
cyclecloud show_cluster slurm-train
cyclecloud show_nodes -c slurm-train
```
`params.json` 예시(핵심 항목):
```json
{
  "Region": "koreacentral",
  "SubnetId": "rg-cyclecloud-training/cc-vnet/compute",
  "MaxHPCExecuteCoreCount": 100,
  "HPCMachineType": "Standard_D4s_v5",
  "SchedulerMachineType": "Standard_D4s_v5"
}
```

## 3.3 생성 확인
- 포털 **Clusters → slurm-train → Arrays** 에서 스케줄러 노드가 `Ready` 인지 확인.
- 스케줄러 노드에 접속해 스케줄러 동작 확인:
  ```bash
  cyclecloud connect scheduler -c slurm-train   # 스케줄러 노드로 SSH
  sinfo    # (Slurm) 파티션/노드 상태
  squeue   # 대기/실행 중인 작업
  ```

## 3.4 클러스터 중지/삭제
- **중지(Terminate)**: 포털에서 클러스터 → **Terminate** (모든 노드 삭제, 설정 유지 → 과금 최소화).
  CLI: `cyclecloud terminate_cluster slurm-train`
- **삭제(Delete)**: 포털에서 클러스터 → **Delete** (설정까지 제거).
  CLI: `cyclecloud delete_cluster slurm-train`

> 💰 실습이 끝나면 **반드시 Terminate** 하여 계산 노드 과금을 막으세요.
