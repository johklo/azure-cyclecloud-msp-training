# 2. CycleCloud 포털 사용 방법

이 문서는 **CycleCloud 웹 포털** 접속 방법, 최초 설정 마법사, GUI 주요 화면 구성, 버전 확인 방법, 그리고 CLI 와의 연동 구조를 설명합니다.

---

## 2.1 포털 접속 및 브라우저 경고 안내

브라우저 주소창에 아래 URL을 입력하여 접속합니다.

```
https://cc-cyclecloud-ekwphu.koreacentral.cloudapp.azure.com
```

> ⚠️ **자체 서명(Self-signed) 인증서 경고**  
> 배포 직후 포털은 자체 서명 TLS 인증서를 사용하므로 브라우저에 **"연결이 비공개로 설정되어 있지 않습니다"** 경고가 나타납니다.  
> **[고급] → [계속 진행]**을 클릭하여 접속하면 됩니다. (운영 환경에서는 정식 인증서 등록 권장)

---

## 2.2 최초 1회 Site 초기화 마법사

서버를 최초 배포한 뒤 첫 접속 시 사이트 초기화 마법사(Setup Wizard)가 실행됩니다.

1. **Welcome / License Agreement**:
   - CycleCloud 사용 약관 동의
2. **Site 이름 & 애플리케이션 관리자 계정 생성**:
   - **Site Name**: 예) `KR-Training`
   - **Admin Account**: 포털 로그인에 사용할 관리자 ID/비밀번호 입력 (Azure 계정과 별개의 CycleCloud 전용 계정)
3. **Azure Subscription (구독) 등록**:
   | 항목 | 입력 값 | 설명 |
   |------|---------|------|
   | **Name** | `training-sub` | 구독 별칭 |
   | **Subscription ID** | `<YOUR_SUBSCRIPTION_ID>` | Azure 구독 ID |
   | **Credentials** | **Managed Identity** 선택 | VM 시스템 할당 관리 ID 사용 (비밀번호/시크릿 입력 불필요) |
   | **Default Region** | `Korea Central` | 기본 노드 생성 리전 |
   | **Resource Group** | `rg-cyclecloud-training` | 노드가 생성될 리소스 그룹 |
   | **Storage (Locker)** | `cclkekwphusd3i` (컨테이너 `cyclecloud`) | 프로젝트/템플릿 보관용 스토리지 |
4. **Save**: 설정 저장 후 클러스터 생성 가능 상태로 전환됩니다.

## 2.3 포털 화면 구성 및 메뉴

![CycleCloud 버전 및 UI 화면](images/add-node/1780364891805.png)

로그인 후 상단/좌측 메뉴에서 아래 기능을 사용할 수 있습니다.

| 메뉴 | 설명 및 기능 |
|------|--------------|
| **Clusters** | 클러스터 목록 확인, 신규 클러스터 생성(`+`), 시작(Start), 중지(Terminate), 삭제(Delete) |
| **Nodes / Arrays** | 노드 배열별 개별 노드 상태(Ready, Acquiring, Error), 로그 확인 및 수동 Scale 제어 |
| **Settings** | 구독(Subscriptions), Locker(Storage), 계정 및 사용자 권한(Users/Groups) 관리 |
| **Marketplace** | 빌트인 클러스터 템플릿(Slurm, OpenPBS, LSF, HTCondor 등) 선택 |

## 2.4 CycleCloud 및 Slurm 버전 확인 방법

KT 운영 지원 시 클러스터의 정확한 버전 정보 확인이 필수적입니다.

### 1) CycleCloud Server 버전 확인
포털 우측 상단의 **`?` (도움말)** 아이콘을 클릭하여 설치된 CycleCloud Server 버전을 확인합니다.

![CycleCloud 버전 확인](images/add-node/1780364891805.png)

### 2) Slurm 버전 확인
**Clusters → 해당 클러스터 선택 → Edit → Advanced Settings** 의 **Slurm Settings** 항목에서 지정된 Slurm 버전을 확인합니다.

![Slurm 버전 확인](images/check-version/1780365604255.png)

## 2.5 CLI (`cyclecloud`) 연동 및 초기화

웹 포털에서 수행하는 모든 작업은 CLI 명령어로도 동일하게 실행할 수 있습니다.

CLI는 CycleCloud Server VM(`cc-server`)에 기본 설치되어 있습니다.

```bash
# SSH 접속 또는 Azure Portal '명령 실행' 이용
ssh -i keys/cyclecloud_rsa azureadmin@20.196.213.145

# CLI 인증 초기화 (최초 1회)
cyclecloud initialize
# -> CycleCloud URL: https://localhost (또는 서버 IP)
# -> Username / Password 입력

# Locker 및 클러스터 목록 확인
cyclecloud locker list
cyclecloud list_clusters
```
