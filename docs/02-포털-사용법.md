# 2. CycleCloud 포털 사용 방법

## 2.1 포털 접속
브라우저에서 접속:
```
https://cc-cyclecloud-ekwphu.koreacentral.cloudapp.azure.com
```
- 자체 서명 인증서라 브라우저 경고가 표시됩니다 → **고급 → 계속 진행**.
- 운영 환경에서는 정식 TLS 인증서를 등록하세요.

## 2.2 최초 1회 설정 마법사 (Site 초기화)
서버를 처음 배포하면 포털 첫 접속 시 **설정 마법사**가 나타납니다.

1. **Welcome / License** — CycleCloud 사용 약관에 동의합니다.
2. **Site 이름 & 관리자 계정 생성**
   - Site Name: 예) `KR-Training`
   - CycleCloud **애플리케이션 관리자** 계정(사용자명/암호)을 생성합니다.
     (⚠️ 이 계정은 Azure 계정과 별개인 CycleCloud 포털 로그인 계정입니다.)
3. **Subscription(구독) 등록** — 아래 값으로 설정:
   | 항목 | 값 |
   |------|-----|
   | Name | `training-sub` (임의) |
   | Subscription ID | `<YOUR_SUBSCRIPTION_ID>` (예: `az account show --query id -o tsv`) |
   | Credentials | **Managed Identity** 선택 (VM 관리 ID 자동 사용) |
   | Default Region | `Korea Central` |
   | Resource Group | 신규 노드용 RG (예: `rg-cyclecloud-training` 또는 자동 생성) |
   | Storage(Locker) | 스토리지 계정 `cclkekwphusd3i`, 컨테이너 `cyclecloud` |
4. **Save** — 이제 클러스터를 만들 수 있습니다.

> 관리 ID 를 쓰므로 시크릿/암호를 입력할 필요가 없습니다. "Validate" 시
> 권한 오류가 나면 5장의 RBAC 확인 절차를 참고하세요.

## 2.3 포털 화면 구성
로그인 후 좌측 메뉴(또는 상단 탭):

| 메뉴 | 설명 |
|------|------|
| **Clusters** | 클러스터 목록/생성/시작/중지/삭제, 노드 현황 |
| **Nodes / Arrays** | 개별 노드(스케줄러/실행) 상태, 시작·종료, 로그 |
| **Settings** | 구독, Locker(스토리지), 사용자/권한, 클라우드 자격 |
| **Marketplace / Templates** | 스케줄러 템플릿(Slurm, OpenPBS 등) |

### 클러스터 상세 화면에서 할 수 있는 것
- **Start / Terminate**: 클러스터 전체 시작/종료
- **Edit**: VM 종류, 노드 수(Max Cores), 네트워크, 스토리지 등 설정 변경
- **Scale**: 노드 배열(node array)별 노드 수 증감 (4장)
- **Arrays 탭**: 각 노드 배열의 실행 노드 목록, 개별 노드 상태/로그
- **Connect**: 특정 노드 SSH 접속 명령 안내

## 2.4 포털 vs CLI
같은 작업을 **포털**(GUI)과 **CLI**(`cyclecloud`) 양쪽으로 할 수 있습니다.
- 학습/일회성 작업 → **포털** 권장
- 반복/자동화 → **CLI** (5장, 각 장의 "CLI" 절 참고)

CLI 는 서버에서 실행합니다. SSH 또는 `az vm run-command` 로 접근하세요(5장).
CLI 최초 사용 시 인증 초기화:
```bash
cyclecloud initialize   # 포털 URL/계정으로 CLI 인증 (최초 1회)
cyclecloud locker list  # Locker 확인
```
