# 9. 파티션 관리 및 신규 파티션 추가

사용 중인 파티션의 노드를 공유하여 새로운 파티션을 추가하는 방법과, 수동 설정이 초기화되는 주의사항을 정리합니다.

---

## 9.1 파티션 설정

스케줄러 노드에서 `azure.conf` 파일을 수정합니다.

```bash
sudo vi /etc/slurm/azure.conf
```

기존 HPC 파티션 설정을 복사한 뒤 `PartitionName` 만 변경하여 추가합니다.

```ini
# /etc/slurm/azure.conf

# 기존 hpc 파티션
PartitionName=hpc Nodes=cycle-lab-hpc-[1-3] Default=YES DefMemPerCPU=1536 MaxTime=INFINITE State=UP
Nodename=cycle-lab-hpc-[1-3] Feature=cloud STATE=CLOUD CPUs=2 ThreadsPerCore=1 RealMemory=3072

# 추가한 new 파티션 (hpc 노드를 공유)
PartitionName=new Nodes=cycle-lab-hpc-[1-3] Default=NO DefMemPerCPU=1536 MaxTime=INFINITE State=UP
```

설정을 반영합니다.

```bash
sudo scontrol reconfigure
```

![azure.conf 수정 및 scontrol reconfigure](images/add-new-partition/1780279495319.png)

> **GUI Max 노드 수정이 필요한가?**
>
> - **기존 노드를 공유하는 파티션**(위 예시): 같은 nodearray의 VM을 빌려 쓰므로 **GUI 수정 불필요**.
> - **전용 VM을 갖는 새 nodearray 파티션**: 자동확장 한도가 nodearray 단위이므로, GUI(또는 템플릿)에서 해당 배열의 **Max Cores / Max VMs** 를 지정해야 합니다([9.5](#95-영구-파티션-추가-템플릿-기반-권장)).

## 9.2 단일 노드 제거 및 생성 시 동작 확인

`hpc` 노드 3대 중 1대를 제거한 뒤 다시 resume하여 정상 동작을 확인합니다.

![단일 노드 제거 화면](images/add-new-partition/1780279503673.png)

```bash
azslurm resume --node-list cycle-lab-hpc-1
```

노드 1대가 정상적으로 다시 기동되는지 확인합니다.

## 9.3 파티션별 Job 실행

파티션은 논리적으로 나뉘었지만 물리 VM은 공유하므로, 물리 자원(코어/메모리)의 제약을 받습니다.

예를 들어 2 코어 VM 3대(총 6 코어) 환경에서 `--cpus-per-task=1` 로 Job을 실행하면, 최대 **6개 Job만 동시 수행**됩니다.

![Job 제출 및 자원 점유](images/add-new-partition/1780279514636.png)

Job을 7번 제출하면 6개가 `RUNNING`, 1개는 `PENDING(Resources)` 상태로 대기합니다.

![7번째 Job 대기 상태 확인](images/add-new-partition/1780279525222.png)

## 9.4 주의사항: 수동 파티션의 자동 초기화

![azslurm 실행 시 GUI 설정으로 초기화](images/add-new-partition/1780279542584.png)

> 🚨 **수동 편집한 파티션은 초기화될 수 있습니다.**
>
> `/etc/slurm/azure.conf` 를 수동 편집해 추가한 파티션은 아래 조건에서 **GUI 설정 템플릿으로 원복되어 삭제**됩니다.
>
> 1. 클러스터를 **Terminate 후 Start** 하는 경우
> 2. `azslurm partition` 또는 `azslurm scale` 명령이 수동/자동으로 실행되는 경우

```bash
azslurm partition
azslurm scale
```

## 9.5 영구 파티션 추가 (템플릿 기반, 권장)

9.1의 수동 편집은 재시작·`azslurm scale` 시 사라집니다(9.4). 영구적으로 파티션을 추가하려면 **클러스터 템플릿**에 `[[nodearray]]` 를 정의하고 재적용합니다.

```bash
cyclecloud export_parameters <클러스터명> > params.json
# slurm.txt 에 [[nodearray specialgpu]] 등 추가 후 강제 재적용
cyclecloud import_cluster <클러스터명> -c slurm -f slurm.txt -p params.json --force
sudo azslurm scale     # 재시작 없이 반영
```

- **파티션 이름 커스터마이즈**: nodearray 이름과 파티션 이름을 다르게 하려면, nodearray의 `[[[configuration]]]` 에 `slurm.partition = my-name` 을 지정합니다.

- **동적(Dynamic) 파티션**: `slurm.autoscale = true` 와 `slurm.dynamic_config` 로 한 파티션에서 **여러 VM 크기**를 Feature로 구분해 사용할 수 있습니다.
